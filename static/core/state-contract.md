# 共享状态契约

本文件定义 `.skill-state/` 下共享状态文件的职责边界，避免重复写、冲突写、无边界写。

## 1. 状态文件列表

### `decision_log.md`

记录会影响后续步骤的结构性决策，例如：

- 目标论文类型与目标体裁
- 术语主映射变更
- 证据分级例外
- 是否允许 direct-entry 跳步
- 写作范围与语言策略

不记录：

- 临时命令输出
- 一次性报错堆栈
- 单个拼写修复

### `error_log.md`

记录可复用的错误模式与修复规则，例如：

- 检索失败的降级路径
- 下载会话异常
- Zotero MCP 连接问题
- 引用审计常见失败模式

不记录：

- 已知且无复用价值的一次性失败

### `term_aliases.md`

记录项目级术语标准化映射：

- Main Term
- Alias
- 使用边界
- 禁用写法

不记录：

- 一般同义词词典
- 与当前项目无关的领域术语

## 2. 写入规则

- Step 1-2 主要写 `decision_log.md`
- Step 2/7/8 可写 `term_aliases.md`
- 任何 Step 在识别出可复用故障模式时可写 `error_log.md`
- 默认只追加高价值、可复用、能影响后续行为的内容

## 3. 读取规则

- Step 1-8 启动前都应读取 `decision_log.md`
- 涉及术语一致性的 Step 2/7/8 应读取 `term_aliases.md`
- 涉及外部工具、下载、Zotero、审计的 Step 应读取 `error_log.md`

## 4. 与工件的边界

`.skill-state/` 只保存共享规则，不替代每一步的正式产物。

正式产物应继续放在用户工作目录中，例如：

- `研究主题.md`
- `大纲关键词.md`
- `检索报告.md`
- `文献-Zotero架构对照.json`
- `综述矩阵.csv`
- `论文初稿.md`
- `引用审计报告.md`
