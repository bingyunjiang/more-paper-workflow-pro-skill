# More Paper Workflow Pro Skill — 功能介绍

> 版本 v1.0.17 | 2026-06-25 | 后续将补充演示案例成果

---

## 一句话定位

**一套围绕真实文献工作的学术写作 Skill。** 它把论文写作拆成 8 个可见、可检查、可回溯的步骤——定题、大纲、检索、筛选、下载、Zotero 管理、写作、润色审计。每一步都有文件化产物，每一条引用都能追溯到真实 PDF 或 Zotero 条目，每一个关键判断都要求回答：证据在哪里？

---

## 这个 Skill 帮你做什么

### 1. 把模糊的研究兴趣，收敛成可执行的研究题目

你不会只有一个"大概方向"就开始检索。Skill 会先判断你当前处于哪个研究阶段——是刚开始摸索、导师给了题目、还是已有初稿想继续深化——然后按阶段引导你走完选题聚焦。

它会在选题阶段就帮你做**预检索**：看看这个方向的文献量、趋势、代表性论文，判断这个题有没有足够的文献底子。产出的 `研究主题.md` 不是简单的头脑风暴笔记，而是一份包含研究对象、研究问题、方法论、创新点初判和风险提示的结构化文档。

选题结束前，还有一道**选题预审**：从原创性、重要性、可行性、文献支撑和方法准备度五个维度打分，告诉你这个方向是绿灯继续、黄灯调整、还是红灯建议换方向。

做完这一步，你知道的不只是"我研究什么"，而是"为什么是这个题、这个题能不能做、怎么把它拆成可检索的关键词"。

---

### 2. 不是"想想大纲"，而是生成可检索、可写作的结构骨架

大纲不只是标题列表。Skill 生成的每章大纲都包含：章节目标、核心论点、需要的证据类型、预期图表和潜在风险。同时会生成一份**关键词清单**——核心词、同义词、上位词、下位词、方法词、场景词——这些词直接对接后续的检索方案。

大纲生成后有**结构化评审**：逻辑连贯性、结构平衡性、创新区分度、工程可行性、格式完备性五项逐个打分，按 P0 必须改到 P3 可选的优先级排序。还会从**导师视角**追问：工作量够不够？有没有 Plan B？能按时毕业或投稿吗？

一套术语别名映射也在这步建立，贯穿后续所有步骤——你在检索时不会因为同一概念用了不同名字而漏掉文献，写作时也不会前后术语不统一。

---

### 3. 检索不是"写个关键词扔进搜索引擎"

Skill 的检索方案是**结构化**的：每个子课题拆成独立的检索任务，每项任务绑定首选数据库、回退数据库和兜底数据库，形成 T1→T2→T3 三级路由。不同学科（医学/工程/CS/综述/中文）有各自的路由策略。

真正检索时，它同时走 Semantic Scholar、Crossref、OpenAlex 三条线，结果按 DOI 去重合并。CS/AI 方向自动补 arXiv，综述类自动做引文扩展。每篇文献经过**五维评分**（主题匹配、方法一致、来源质量、时效性、引用量），分为 T1 必下必读、T2 尽量下载、T3 候选参考、T4 剔除四档。

检索完成后你会得到：一份 PRISMA 风格的可视化检索报告、一份含摘要的检索文献表（.md + .xlsx + .pdf + .bib）、一个饱和度分析判断是否继续检索、一个本地 HTML 检索结果看板。**你很清楚每一篇文献为什么入选、排在什么位置、证据可信度是什么级别。**

---

### 4. 统一下载路由：一个入口，覆盖 24 家出版社

这是整套 Skill 最"硬"的工程模块。你不需要知道每篇论文在哪家出版社、该用什么方法下载。把 DOI、标题、BibTeX 或参考文献列表给 `unified_download_router.py`，它会自动识别出版商，分发给对应的下载策略。

具体怎么做：先试 Sci-Hub 免费源，不成就走 OA Fast（按 Unpaywall → OpenAlex → Semantic Scholar 依次找公开版本），再不成就启动 CDP 真实浏览器，通过 Chrome DevTools Protocol 操控浏览器完成出版社文章页的 PDF 提取。整个过程自动处理 Cloudflare Turnstile 验证、SSO 机构登录门控、CNKI 验证码等待——你需要机构登录时它会停下来告诉你，登录完它接着跑。

你甚至可以**直接从 Step 5 开始**——如果你已经有 DOI 或参考文献列表，不需要跑前面四步，直接进入下载。没解析清楚的标题会单独列出让你确认，不会悄悄跳过。

支持 24 家出版社，覆盖 Elsevier、IEEE、Springer、Wiley、Taylor & Francis、Sage、ACM、RSC、MDPI、CNKI、万方等主流学术来源。

---

### 5. Zotero 不只是存 PDF，而是可写作的真实资料库

Skill 会根据你的论文大纲**自动生成 Zotero 集合结构**——根集合是论文题目，一级集合对应章节，子集合对应二级目录。然后结合检索结果、BibTeX 和 PDF 附件池，生成一份 `文献-Zotero架构对照`：每篇文献该归入哪个集合、打什么标签、PDF 有没有、附件缺不缺，一目了然。

英文文献用 BibTeX/DOI 入库，中文文献（CNKI/万方）不用虚构 DOI，而是用 `source_id` + 详情页 URL + 中文元数据生成 CSL JSON——每篇文献都有可追溯的来源。

入库完后做**三层核对**：集合结构对不对、条目齐不齐、PDF 附件关联上没有。缺附件、错集合、重复条目全部单独列出。最后生成一份能力索引，告诉你当前文库能支撑哪些章节的写作，哪些地方还缺材料。

这一步的意义在于：当 Step 7 开始写作时，你不需要在几十篇 PDF 里大海捞针，而是按章节定位到对应的 Zotero 子集合，直接在精读完的文献上写作。

---

### 6. 写作从证据出发，而不是从"文笔"出发

这是这个 Skill 和普通 AI 写作工具最根本的区别。它不是"输入标题 → 生成正文 → 随便补几个引用"，而是**先建证据层，再在证据边界内写作**。

写作启动时，它先做三件事：
- **文献证据矩阵**：13 列结构化提取——每篇文献的研究问题、方法、核心发现、局限、和你研究的关系、可直接引用的摘录——逐级回退：Zotero 笔记优先，其次 PDF 标注，再次元数据，最后才读全文。
- **目标体裁学习**：你指定要投的期刊或要写的论文类型，Skill 分析其标题风格、段落密度、图表惯例、语言惯例，生成风格配置和章节蓝皮书。
- **章节论证计划**：在动笔之前锁定这一章的核心论点、需要的证据、预期的图表，以及"如果缺证据怎么办"的回退策略。缺证据就先不硬写。

正文写作时，它按 Zotero 子集合逐节取证——只读当前小节对应的 PDF，不扫整个文库。直接读 PDF 原文而非向量分块，确保引用的每一个数据、方法、结论都能在原文里找到出处。试写阶段还会逐条标注（已读全文）/（已读摘要）/（仅元数据），让你清楚每条引用的证据深度。

写作范围灵活：你可以完整写出整篇论文，也可以只写综述章节、只写指定章节、续写已有草稿、只写摘要，或者仅按审稿意见做定向修订。

---

### 7. 写完不是结束，引用审计才是收口

正文完成后，Skill 会对每一条引用做**三层审计**：

- **格式对不对**：GB/T 7714-2015 完整规范，中英文排序、作者格式、7 种文献类型代码、缺失元数据的正确处理方式。
- **映射对不对**：正文里的"[Author, Year]"能不能追溯到真实的 BibTeX 条目或 Zotero 记录？有没有引用了一篇根本不在文库里的文献？
- **证据撑不撑得住**：被引文献是不是真的说了正文里那句话？有没有用摘要级证据支撑了强因果判断？

每条引用的审计结论都附**动作建议**：保留、降级表述、补充 PDF 或全文证据、修复条目映射、或者直接替换/移除。高风险引用不能带入定稿。

收到审稿意见后也有标准流程：把审稿意见拆成修稿路线图、回应骨架和证据缺口清单，修稿后做复评验证问题是否真正关闭、有没有引入新问题、引用风险是否下降。

---

### 8. 润色只做该做的，不做不该做的

Step 8 的定位很克制：它是**诊断优先的保守修订**，不是重写。

它会先把问题分成五类——证据缺口、结构漂移、机械化表达、贡献过度声称、引用错配——然后再判断：哪些可以直接修（术语统一、语言清理、论断降强度、补桥接句），哪些需要作者自己决定，哪些依据不足不能动。

不会做的事同样重要：不新增外部证据或引用、不重写章节主体、不把自己的文风偏好设成硬标准。

它还有一个内置的 AI 味诊断子层，专门识别机械化表达、空泛套话、句长节奏过匀和格式层脏污——但只服务润色诊断，不会因为"像 AI"就否定稿件。

最终输出三档判决：ready for finalize / ready with warnings / not ready requires rollback——让你明确下一步是该定稿、还是回退修稿、还是回去补资料。

## 质量保障不是一道墙，是一张贯穿全流程的网

这个 Skill 把质量控制分散在每一步里，而不是堆在最后。8 道防线从前到后逐层收紧：

选题预审（Step 1）判断方向值不值得做 → 大纲评审（Step 2）修结构不把问题推到正文 → 文献评分（Step 4）确认每一篇该不该进入文库 → 章节论证计划（Step 7）在动笔前锁定证据边界 → 段落自查和引文支撑（Step 7）确保每段有明确职责和足够证据 → 修稿复评（Step 7）验证改过的稿子问题到底关没关 → 引用审计（Step 7）逐条核验格式、映射和证据 → 诊断润色（Step 8）终稿前最后一遍风险收口。

核心逻辑很简单：**越早发现问题，修复成本越低；越靠近终稿，越要区分能就地修还是必须回退。**

## 对你来说意味着什么

**如果你是研究生或博士生**，在写毕业论文或综述论文：这个 Skill 能帮你从选题一路走到终稿，每个阶段都留下可检查的中间产物。你的导师问你"这个数据哪来的"时，你能在 30 秒内把原文 PDF 翻出来。

**如果你已经在用 Zotero**，但文库整理和写作是两件事——检索完了不知道怎么把文献系统性地组织起来，组织好了不知道怎么写进去：Step 6+7 的组合就是为你设计的，从结构化索引到证据矩阵到正文按节输出。

**如果你被 AI 编造参考文献坑过**，不敢再用 AI 写论文：这个 Skill 的整个链路都是围绕"真实文献"设计的——文献先进入 Zotero 或证据包，再从证据往外写，写完还要走引用审计。AI 负责提效，真实文献负责兜底。

**如果你有 DOI/BibTeX/参考文献列表**，只想快速批量下载 PDF：你可以直接从 Step 5 进入，不需要跑前面的步骤。24 家出版社一个入口，自动路由、自动绕过反爬、自动处理登录。

## 工程层面

- **跨平台**：macOS / Windows / Linux 完整支持，Python CLI 优先，不依赖 bash
- **多宿主**：同一套 Agent 规则在 Claude Code / Codex / Hermes / OpenClaw 上运行
- **可回溯**：每一步都有 Completion Gate——产物没齐、证据没够，就不能说"这步完成了"
- **可恢复**：下载中断不重跑已完成篇目，登录门控支持 checkpoint 恢复
- **自检**：启动时自动检查依赖和版本更新，缺失即打印安装指引
