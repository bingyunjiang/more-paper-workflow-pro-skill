# 电力电子 / 能源系统目标体裁风格包

本文件只在目标期刊、会议、中文核心或学位论文体裁明确时加载。它不得作为所有电力电子/能源论文的全局默认风格，也不得覆盖证据等级、claim 强度或用户指定格式。

## 使用边界

- 未指定目标期刊/体裁时，只使用 `power-electronics-ev-energy-writing.md` 的工程证据规则。
- 目标风格只能约束结构密度、图表呈现、参数完整性、讨论深度和边界说明。
- 如果目标期刊风格与当前证据等级冲突，以证据等级为准。

## 1. IEEE Transactions / IEEE Access 类

适合：电力电子拓扑、控制、并网、EV charging、V2G、无线电能传输、储能系统。

写作重点：

- 摘要直接说明问题、方法、关键量化结果和验证平台。
- 引言按 `application need -> technical gap -> proposed method -> contributions` 推进。
- 方法必须给出可复现控制框图、关键方程、参数表和实验/仿真设置。
- 结果应把波形、效率曲线、热结果、稳定性分析和对比表绑定到具体 claim。
- 贡献点避免泛泛写 `novel / efficient / robust`，要落到拓扑、控制、损耗、动态性能或系统约束。

常见风险：

- 只有仿真没有硬件验证，却按实验论文口径写。
- 图表很多但没有逐一回答 claim。
- 缺参数表、器件信息、采样频率或控制参数。

## 2. Applied Energy / Energy 类

适合：充电站能量管理、储能配置、V2G 经济性、能源系统优化、低碳调度。

写作重点：

- 明确系统边界：站级、区域级、微电网、配网、交通-电网耦合。
- 明确指标：成本、碳排、峰谷差、弃电、负荷率、用户等待、退化成本。
- 场景设计要有数据来源、时间粒度、边界条件和基准策略。
- 优化或 AI 方法必须说明约束、求解器、训练/测试划分、消融或灵敏度分析。
- 讨论中区分技术结果、经济结果、政策/市场假设。

常见风险：

- 只报告收益百分比，不说明电价、车流、负荷和储能容量。
- 将单一场景收益外推为普遍结论。
- 忽略电池退化、用户可用性和并网容量。

## 3. 工程中文核心

适合：充电桩工程应用、电力电子装置、控制策略、储能系统、无线充电实验平台。

写作重点：

- 术语、参数、图表编号和单位必须稳定。
- 方法章节优先交代工程对象、实验平台、控制策略和测试条件。
- 结果章节重视参数完整、图表规范和工程边界，不宜过度理论化外推。
- 讨论可简洁，但必须说明适用范围、局限和后续工程化问题。

常见风险：

- 参数表缺失，导致实验不可复现。
- 图中曲线、工况、单位和图注不完整。
- 结果口径从“样机/仿真场景”跳到“大规模应用”。

## 4. 学位论文电力电子 / 能源系统章节

适合：硕士/博士论文中的系统架构、控制策略、实验平台、仿真与工程验证章节。

写作重点：

- 章节深度应按 `工程场景 -> 系统约束 -> 方法/模型 -> 参数设置 -> 仿真/实验 -> 对比 -> 边界` 展开。
- 控制章节要区分控制目标、模型假设、稳定性证据和参数设计。
- EMS 章节要区分预测、优化、调度、执行和评价指标。
- V2G / 储能章节要同时讨论电网侧、用户侧、设备侧和经济侧约束。
- 仿真和实验必须分清，不能把仿真平台写成工程验证平台。

常见风险：

- 综述像教材，缺少当前论文问题链。
- 方法章节缺参数表和变量定义。
- 结果章节只贴波形，不解释指标、工况和边界。

## 输出约束

当使用本文件时，Step 7 的 `style_profile` 至少记录：

- `target_venue_or_genre`
- `required_parameter_detail`
- `figure_table_density`
- `reproducibility_expectation`
- `discussion_depth`
- `overclaim_risks`

目标体裁风格只能帮助组织表达；所有 claim 仍必须通过 `required_evidence`、`evidence_anchor` 和 `support_grade` 判断。
