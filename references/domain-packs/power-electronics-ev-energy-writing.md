# 电力电子 / 充电 / V2G / 能源系统写作领域包

本领域包只在任务命中充电桩、储能、电力电子、EMS、V2G、快充、无线充电、超级电容、并网变换器、充放电控制或能源调度时加载。它不是材料/机械论文默认规则，也不替代 Step 7 的引用审计和图表证据绑定。

## 触发范围

- 中文：充电桩、储能、电力电子、能量管理、EMS、V2G、V1G、V2H、V2B、快充、无线充电、超级电容、并网、双向变换器、DAB、LLC、整流器、逆变器、微电网。
- English: `power electronics`, `EV charging`, `energy storage`, `EMS`, `V2G`, `fast charging`, `wireless charging`, `supercapacitor`, `bidirectional converter`, `grid-connected`, `microgrid`.

## 1. power_topology_card

用于拓扑、硬件平台、功率级对比和功率密度相关段落。

必须交代：

- 拓扑结构：AC/DC、DC/DC、LLC、DAB、Vienna、三电平、矩阵变换器、双向变换器等。
- 关键参数：功率等级、输入/输出电压范围、母线电压、开关频率、磁件、器件类型、冷却方式。
- 对比边界：同功率等级、同输入/输出范围、同散热条件、同负载点或同体积/质量口径。

高风险写法：

- 只给“高效率/高功率密度”，不交代功率等级、测试条件或体积/质量基准。
- 不说明对比对象工况是否一致。

## 2. control_and_stability_card

用于控制策略、动态响应、鲁棒性和稳定性段落。

必须交代：

- 控制对象：电压、电流、功率、SOC、频率、相位、阻抗、温度或能量流。
- 控制方法：PI、MPC、droop、PLL、滑模、预测控制、强化学习、规则控制等。
- 稳定性证据：小信号模型、Bode/Nyquist、Lyapunov、阻抗稳定、仿真扰动、硬件阶跃、HIL 或实验波形。
- 鲁棒性边界：负载变化、温度、器件参数、采样频率、通信延迟、电网扰动。

高风险写法：

- 只给仿真波形就写“系统稳定”。
- 只写“鲁棒性强”，没有参数扰动、边界条件或对比基准。

## 3. efficiency_loss_thermal_card

用于效率、损耗分解、热管理和可靠性相关段落。

必须交代：

- 损耗来源：开关损耗、导通损耗、磁损、驱动损耗、辅助电源、散热损耗。
- 效率测量：仪器、负载点、功率等级、温度、输入/输出电压范围、峰值与全负载曲线。
- 热管理：环境温度、冷却方式、热阻、温升、稳态/瞬态热结果、热像或传感器位置。

高风险写法：

- 用峰值效率替代全负载效率图。
- 用仿真效率替代实测效率却不标注。
- 温升结果没有环境温度、稳态条件或测点说明。

## 4. EMS_optimization_card

用于充电站能量管理、储能调度、车网协同和优化算法段落。

必须交代：

- 目标函数：电费、峰谷差、碳排、变压器负载、储能寿命、用户等待时间、排队长度。
- 约束：SOC、功率、电价、车流、储能容量、并网功率、用户行为、通信延迟、设备寿命。
- 基准：无 EMS、规则控制、传统优化、已有算法或工程策略。
- 可复现：数据来源、时间粒度、场景数量、求解器、随机种子、训练/测试划分。

高风险写法：

- 只给总成本下降，不说明电价、负荷、车流和约束。
- 强化学习或 AI 方法没有基准、消融或泛化场景。

## 5. V2G_grid_claim_card

用于 V2G 并网服务、电网友好性、聚合调度和市场收益段落。

必须交代：

- 功率流类型：V1G / V2G / V2H / V2B / V2X。
- 服务类型：削峰填谷、频率响应、备用、无功支撑、电压调节、需求响应。
- 电池影响：循环次数、SOC 窗口、退化成本、用户可用性、车辆停留时间。
- 并网约束：逆变器容量、通信、安全、聚合商、市场规则、并网标准。

高风险写法：

- 只写“V2G 提升电网稳定性”，没有电网指标、控制策略或场景边界。
- 只写收益，不写电池退化、用户约束和并网容量。

## 6. wireless_fast_charging_card

用于快充、无线充电和超级电容章节。

必须交代：

- 快充：倍率、充电曲线、温升、寿命影响、安全边界、电池类型。
- 无线充电：耦合系数、偏移容忍、补偿拓扑、效率、EMI/EMC、异物检测。
- 超级电容：功率密度、能量密度、循环寿命、ESR、热表现、混合储能分工。

高风险写法：

- 只比较充电时间，不写热、安全和寿命边界。
- 无线充电只给最大效率，不给偏移、负载范围和EMI/EMC边界。

## 工程证据模态

涉及工程机制、系统性能或图表 claim 时，`mechanism_cards`、`figure_table_panel_binding` 和引用审计可以使用以下 `evidence_modality`：

```text
waveform / efficiency_curve / loss_breakdown / thermal_map / grid_metric / optimization_result / hardware_prototype / HIL / field_data / standard / simulation / user_data
```

这些模态只说明证据类型，不自动提高证据等级。仿真、HIL、样机实验、现场数据和标准文件必须在正文中区分。

## 降强度规则

| 缺失项 | 允许写法 | 禁止写法 |
|---|---|---|
| 无硬件实验 | 仿真结果提示该策略在设定工况下具有潜力 | 已验证工程可行性 |
| 无稳定性分析 | 波形显示该场景下响应可接受 | 系统稳定/鲁棒性强 |
| 无损耗分解 | 测得效率在某负载点较高 | 损耗机制得到优化 |
| 无退化成本 | V2G 可能带来调度收益 | V2G 显著提升综合收益 |
| 无偏移测试 | 对准工况下无线充电效率较高 | 无线充电系统具有高鲁棒性 |

## Step 7 / Step 8 边界

- Step 7 使用本领域包生成工程 claim 的证据需求、图表绑定、边界条件和 `evidence_gap_list`。
- Step 8 只识别工程空话、降强度、补边界句或提示回退，不新增实验、标准、仿真或引用。
