# 大纲 Story Shapes

本文件用于 Step 2 大纲设计，不等同于期刊级 abstract 模板。

## 目标

把大纲从“章节列表”升级为“论证路径”。

## 常用故事形状

### 1. 问题 -> 原因 -> 方案 -> 验证

适合：

- 工程优化
- 方法论文
- 应用研究

### 2. 背景 -> 谱系 -> 空白 -> 评价框架

适合：

- 文献综述
- 学位论文绪论

### 3. 场景 -> 约束 -> 取舍 -> 实证

适合：

- 工程报告
- 方案设计

### 4. 机制 -> 建模 -> 参数 -> 预测/控制

适合：

- 仿真
- 理论分析
- 机理研究

## 使用原则

- 先选一个主 story shape，再定章节
- 章节标题应服务主 story，不应只是材料堆叠
- 大纲应显式说明每章承担的功能
