# 工程文档→博士论文大纲优化工作流

> 适用于：已有论文大纲草稿 + 一组现有工程技术报告（.docx/.doc），从中提取数据和工程发现来充实、优化大纲。

## 完整工作流（3 Stage）

### Stage 1: 文档组合分析

#### 1a 文档扫描

```python
import os, docx

base = "/path/to/docs"
files = sorted([f for f in os.listdir(base) if not f.startswith('.')])
for f in files:
    size = os.path.getsize(os.path.join(base, f))
    print(f"{f}  ({size/1024:.0f} KB)")
```

#### 1b 文本提取

- **.docx**: `python-docx` → `Document(path).paragraphs`
- **.doc（旧格式）**: `textutil -convert txt`（macOS 内置）
- **提取要点**：前80行（识别结构目录）+ 后30行（识别结论/附录）+ 关键数据段落

```python
import docx
def extract_docx_text(path):
    doc = docx.Document(path)
    return [p.text for p in doc.paragraphs if p.text.strip()]

# .doc旧格式用textutil
import subprocess
def extract_doc_text(path):
    outpath = "/tmp/converted.txt"
    subprocess.run(["textutil", "-convert", "txt", "-output", outpath, path],
                   capture_output=True, timeout=30)
    with open(outpath, "r", encoding="utf-8", errors="replace") as f:
        return [l.strip() for l in f if l.strip()]
```

#### 1c 文档关系拓扑构建

建立核心文档 → 支撑文档 → 专题文档 → 验证文档的层级关系：

| 层级 | 特征 | 典型文档 |
|------|------|----------|
| 核心 | 覆盖设计-制造-试验全流程 | 研制总结报告 |
| 设计支撑 | 具体参数计算 | 设计计算报告 |
| 专题深入 | 核心技术创新 | 专项技术说明书 |
| 试验验证 | 测试数据和结论 | 试验总结报告 |
| 改进迭代 | 问题整改+换装方案 | 改进研制总结报告 |

#### 1d 技术路线提取

从文档中提取3条核心技术路线及其定量数据：

```
路线A: 方法名称 → 参数/手段 → 定量结果
路线B: 方法名称 → 参数/手段 → 定量结果
路线C: 方法名称 → 参数/手段 → 定量结果
```

### Stage 2: 大纲-文档交叉映射

#### 2a 映射矩阵

逐章列出大纲各节与工程文档的对应关系：

| 大纲章节 | 对应文档 | 可注入数据 | 优先级 |
|----------|---------|-----------|:------:|
| Ch1 背景 | 总体技术报告§1, 改进报告§1 | 产品参数、行业标准 | P1 |
| Ch2 数值方法 | 设计计算报告§3.4 | 网格量、湍流模型、步长 | P1 |
| Ch2 压力脉动 | 设计计算报告§3.5 | 监测点数据、特征值 | P1 |
| Ch3 叶轮优化 | 总体技术报告§3.3-3.8 | 多方案迭代路径 | P0 |
| Ch4 支撑结构设计 | 专项技术说明§3 | 多阶段演化、关键数据 | P0 |
| Ch5 试验验证 | 试验报告全文 | 长周期试验、标准项目 | P0 |
| Ch5 装配工艺 | 总体技术报告§5.3 | 跳动公差、工装架 | P0（新增节） |
| Ch5 实机换装 | 改进报告§15 | 换装工期、隔振数据 | P0（新增节） |

#### 2b 缺口定位

三类缺口及处理：

| 缺口类型 | 含义 | 处理 |
|----------|------|------|
| **跨章节空洞** | 工程文档中有内容但大纲未覆盖 | 新增子节 |
| **数据空缺** | 大纲有节但缺具体数值 | 从文档中提取注入 |
| **连接带空白** | 两章之间缺少过渡内容 | 新增过渡节（如"残余激励→结构设计输入"） |

### Stage 3: 逐章注入与优化

#### 3a 设计参数注入

```markdown
【原大纲】额定流量Q、扬程H
【注入后】额定流量Q m³/h、扬程H m、转速n rpm、比转速ns
```

#### 3b 数值方法具体化

```markdown
【原大纲】数值模拟方法
【注入后】k-ε湍流模型+SIMPLE算法+二阶离散迎风格式+MRF多参考坐标系模型
- 网格：单级~N₁万（叶轮旋转域~N₂万，蜗壳段~N₃万，进出口~N₄万）
- 时间步长：Δt s（叶轮旋转1°），共K周期
- 监测点：叶轮侧P1~P6（R₁ mm），蜗壳侧P7~P12（R₂ mm）
```

#### 3c 试验数据注入

```markdown
【原大纲】试验验证
【注入后】X小时常规运转试验（GB/TXXXX X级）+ 冲击试验（GJB150.18-86）
+ 倾斜摇摆（GJB150.23A-2009）+ 湿热（GJB150.9A）+ 振动（GJB150.16A）
→ 全部通过，拆检无异常
```

#### 3d 叙事线强化

将工程文档中隐含的设计迭代逻辑提炼为叙事线：

- **结构多阶段演化**：初始方案→中间优化→最终方案
- **水力方案多迭代**：方案一(基准)→方案二→…→方案N(最终构型)
- **抑源→隔振双主线**：Ch3降低激励→Ch4阻断传递

### 常见陷阱

1. **.docx 含大量图片导致加载慢**：30MB+文件需5-10s，设置timeout=30
2. **.doc旧格式TOC域代码**：textutil暴露未更新的域代码，不影响正文
3. **重复文件**：用 `os.path.getsize` 预扫描，标记大小相同的文件
4. **中文PDF emoji不支持**：STHeiti无emoji字形，用文字替代（如[是]）
